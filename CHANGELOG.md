## [v0.1.0] - 2026-03-13

### Highlights

- Pure-Go font glyph coverage checker for Pandoc/LaTeX document validation
- Drop-in replacement for fc-list font discovery with structured output

### Added

- `fontscan check` command to verify font supports all characters in a document
- `fontscan list` command to list and filter available fonts with `--filter`, `--family-only`, and `--limit` flags
- `fontscan info` command to display font information with optional `--coverage` for Unicode block analysis
- `fontscan recommend` command to find fonts supporting all characters in a document
- `fontscan pandoc` command to parse YAML frontmatter and validate font settings
- `font` package for loading TTF, OTF, and TTC fonts with glyph coverage analysis
- `scanner` package for cross-platform font discovery (macOS, Linux, Windows)
- `checker` package for document-to-font compatibility validation
- `pandoc` package for Pandoc frontmatter parsing with default font database
- `format` package with TOON (default), JSON, and JSON-compact output formats
- Unicode character names in missing character reports (e.g., "U+2192 RIGHTWARDS ARROW")
- 36 common Unicode block coverage analysis
- Flexible font name matching (case-insensitive, space-insensitive)
- Exit code 1 when missing characters found for CI/CD integration

### Documentation

- README with CLI usage examples and library API documentation
- CLAUDE.md with AI agent instructions and workflow guidance
- fc-list to fontscan migration guide in README
